# Phase 7F Cross-Platform Visual Parity And Intelligence UI Pattern

Status: implemented by gate, screenshots, source checks, and carry-forward audits.

## Locked Reading Rhythm

Every Predicta school surface must follow the same user-value rhythm:

1. Prediction first.
2. Evidence second.
3. Action and guidance third.
4. Safety and limits last.

This rhythm is not a teaching module. It is a product discipline that prevents
Predicta from sounding like an internal system document, a toolkit, or an
astrology classroom before the user receives value.

## School Starts

- Vedic starts with charts, graha/house evidence, then direct life prediction.
- KP starts with the user question or refined event, verdict, timing, and proof.
- Nadi starts with karmic story, validation, activation, and practice.
- Numerology starts with number identity, cycle, rhythm, and practical guidance.
- Signature starts with confirmed visible traits, reflective guidance, and
  safety boundaries.
- Life Atlas starts with life journey, destiny pattern, current chapter, and
  soul-purpose synthesis.

## Parity Rules

- Web, mobile, PDF/report, and chat context must use the same school boundaries.
- Specialist rooms must not drift into each other unless the user chooses Life
  Atlas or a clear handoff.
- Technical details stay preserved, but they cannot appear before the user value.
- Predicta chat must explain what the user sees without changing school method.
- Report samples must not read like internal contracts or astrology lessons.
